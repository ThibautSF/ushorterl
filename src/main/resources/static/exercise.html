1. Création d'une mini API en Java

Le but de cet exercice est de créer de bout en bout une petite API web REST permettant de gérer des "short URL". L'implémentation de cette API devra respecter quelques contraintes techniques :

  • Implémentation en Java 8 ou supérieur
  • Utilisation du gestionnaire de dépendances Maven
  • Utilisation du framework Spring Boot
  • Utilisation de l'IDE IntelliJ IDEA

L'API devra exposer à minima les services REST suivants :

  • POST /short_url permettant de créer et persister une short URL
  • GET /short_url/:short_url_id permettant de récupérer une short URL
  • PUT /short_url/:short_url_id permettant de modifier une short URL
  • DELETE /short_url/:short_url_id permettant de supprimer une short URL

Les short_url devront être utilisables, c'est-à-dire qu'elles doivent permettre de suivre le lien court pour accéder au lien d'origine.

La version d'API livrée devra être hébergée sur un serveur web et accessible publiquement, dans le meilleur des cas sur un projet Google App Engine.

Toute fonctionnalité supplémentaire sera la bienvenue.

2. Documentation de l'architecture et du code

Cette partie consiste à documenter en anglais :

  • dans un document, tous les choix techniques liés à ce mini-projet (architecture, frameworks, etc)
  • dans le code, via des commentaires (ie. type javadoc)

3. Mise à disposition

Mettre à disposition les sources et la documentation sur un repo git publique.

  • préférence pour GitHub
  • possibilité de mettre la documentation demandée dans la partie 2. dans le fichier readme.md à la racine du git
